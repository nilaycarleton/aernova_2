# Design Context

Before changing any user-facing UI, read `PRODUCT.md` (strategy: who this is for and why) and `DESIGN.md` (visual system: colors, type, components).

Aernova is a **product** surface on **web**, built for **non-technical roofing contractors**. The pipeline underneath is photogrammetry; none of that vocabulary belongs in front of the user. Personality is **calm, clear, effortless**. Accessibility target is **WCAG 2.2 AA**.
